# Dia 6 - Controle de Versionamento

## Local e Origin

Uma analogia simples para entendermos o que é o `local` e o `origin` é pensar no local como a Galeria de Fotos do seu celular e o origin como o Google Photos (a nuvem).

## Local (Galeria do Celular)

É o seu computador.

Você tira várias fotos (faz commits).
Você edita, apaga e organiza essas fotos.
Tudo isso acontece só no seu aparelho. Se o seu celular cair na piscina hoje, você perde tudo. Ninguém na internet consegue ver suas fotos ainda.

## Origin (Google Photos)

É o GitHub (a nuvem).

É onde você guarda as fotos para que elas estejam seguras (backup).
É onde outras pessoas podem ver suas fotos, caso você dê permissão.
Origin é apenas o "apelido" padrão que o Git dá para esse lugar na internet de onde você clonou o projeto.

## Sincronizando (Push e Pull)

| Ação             | Comando    | Analogia                                                                                                                                                                                |
| ---------------- | ---------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Fazer Backup     | `git push` | Você envia as fotos da sua Galeria (Local) para o Google Photos (Origin). Agora elas estão salvas na nuvem.                                                                             |
| Restaurar Backup | `git pull` | Você baixa as fotos do Google Photos (Origin) para a sua Galeria (Local). Útil quando você troca de celular (computador) ou quando outra pessoa adicionou fotos no álbum compartilhado. |

## Git amend

O comando git commit --amend é usado para modificar o último commit que você fez. Isso é útil quando você percebe que esqueceu de adicionar algum arquivo ou quer ajustar a mensagem do commit. Ao usar esse comando, o Git combina o novo conteúdo que você especificou com o commit anterior.

## Por que dá erro ao usar push após git commit --amend?

Quando você faz um push para o GitHub, ele tenta enviar a versão do seu repositório local que corresponde à versão do repositório remoto. Quando você usa git commit --amend, você altera o histórico de commits. Isso significa que o commit que você tinha antes não existe mais na mesma forma que estava.

Quando você tenta dar um git push, o Git vê que o histórico do seu repositório local está diferente do remoto e gera um erro, porque ele não pode simplesmente sobrescrever o histórico remoto.

## Lembre-se!

Usar --force pode sobrescrever alterações no repositório remoto que outras pessoas possam ter feito. Portanto, é bom ter certeza de que você é o único trabalhando naquela branch ou que você tem certeza de que ninguém mais fez alterações que você vai perder.
Uma alternativa mais segura ao --force é usar --force-with-lease, que evita sobrescrever alterações que você não conhece.
